Answer guidance

a01 — Everyday algorithms

  • Quick checks: decision = diamond; sequence = ordered steps; pseudocode = structured plain language.
  • Step 2 table: sequence = ordered steps; decision = branch on condition; hard symbol e.g., connector.
  • Task A table: ensure 5 steps, ticked decision where relevant.
  • Task B condition: fill Yes/No paths (e.g., cereal available → eat cereal / toast instead).
  • Flowchart checklist: start/end present; Yes/No labels; loop exit identified.
  • Loop risk: “wait forever” if no condition to exit (e.g., no cereal scenario).

a02 — Inputs and outputs

  • Quick checks: IO symbol = parallelogram; input = data/signals in; output example = heater on.
  • Step 2 table: camera input/output; washer input/output; microwave input/output.
  • Device tables: expect at least 2 inputs/2 outputs per device.
  • Snippet planner: input trigger, optional decision, output action; IO symbol checkbox ticked.
  • Checklist table: input present, output present, test idea written.
  • Device examples: camera (inputs: button, light sensor; outputs: flash, click sound), washing machine (inputs: door sensor, program knob; outputs: heater on, motor spin).
  • Misconception: calling “water” an output—clarify controller outputs vs environment.

a03 — Flowchart for a machine

  • Quick checks: start/end = oval; decision has two exits; exit condition prevents infinite loop.
  • Step 2 table: machine choice + behaviour.
  • IO table: two inputs/two outputs noted.
  • Flowchart element table: condition (e.g., light low), Yes action (flash on), No action (flash off), end = photo saved.
  • Loop check: radio yes/no plus exit (e.g., timeout, max retries).
  • Example camera flash path: Start → Check light → Yes: flash on → Capture → End; No: flash off → Capture → End.
  • Loop exits: timeout on waiting for door closed; max retries on fill/empty; timer reaching zero in microwave.

Evidence to collect

  • Labelled decisions and IO symbols in notes/photos.
  • At least one reflection noting a potential infinite loop and fix.
  • Pseudocode (optional) aligning with the flowchart steps.
