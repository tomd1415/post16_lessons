Lesson plan

Overview

Pupils learn to represent algorithms with flowcharts and pseudocode, focusing on sequence, decisions, and inputs/outputs in real devices.

Timing: ~60 minutes

Resources: Paper/whiteboards, markers, projector, printable IO/device cards.

Learning objectives

  • Define sequence and its purpose in algorithms.
  • Recognise representation methods: flowcharts, pseudocode.
  • Use flowchart symbols: start/stop, process, decision, input/output, arrow.
  • Outline and fix the sequence of operations in a flowchart/pseudocode.

Lesson structure (suggested)

  1. Starter (8m): Symbol recall using quick checks (a01).
  2. Guided practice (15m): Everyday routine with one decision (a01 tasks A–C).
  3. Collaborative (15m): Inputs/outputs identification and flowchart snippet (a02 tasks).
  4. Main task (18m): Machine flowchart (camera/washing machine/microwave) plus pseudocode (a03).
  5. Reflection (4m): One loop risk and exit strategy.

Assessment & evidence

  • Quick check answers in each activity.
  • Step 2 notes defining symbols and IO examples.
  • Flowchart sketches with labelled decisions and IO parallelograms.
  • Optional pseudocode matching the chart.

Differentiation

  • Support: Provide printed symbol legend; give a starter flowchart with missing labels.
  • Stretch: Add a loop with a timeout; ask for two alternative paths for the machine.
  • Misconceptions: missing End nodes; unlabelled Yes/No exits; confusing environment input with controller output.
