a04 - Specification for a Game

Goal: Write a concise specification for a simple game, separating goals, rules, inputs/outputs, and constraints.

Step 1: Quick checks

Prompt 1: A good specification should be...

Prompt 2: Functional requirements describe...

Prompt 3: A non-functional requirement for a game is...

Step 2: Key ideas in your own words

Step 3: Build the main task

Learning objective: Understand the terms program description and specification.

Task A: Choose a game idea

Pick a simple concept (e.g., “collect the stars”, “dodge the asteroids”, “quiz of 10 questions”).

Task C: Add acceptance checks

Write 3 short “The game should…” statements that can be tested.

Stretch: add one accessibility requirement (e.g., colour contrast, keyboard-only controls).

Step 4: Reflection

Which requirement is hardest to test? What would you show a teammate to prove the game meets the spec?

Teacher guidance (answer prompts)

  • Quick checks: (1) clear/testable, (2) functional = what it does, (3) non-functional = performance/quality.
  • Look for “acceptance” statements that are measurable; discourage vague adjectives without tests.
  • Misconceptions: confusing goals with constraints; listing UI colours as functional; missing accessibility.
  • Differentiation: provide a starter spec template; stretch to include accessibility and performance numbers.
