How this works: This activity is broken into small steps with quick checks and spaces to add your own ideas.

Step 1: Quick checks

Prompt 1: Which example is a formal language?

Prompt 2: Why do computers prefer formal language?

Prompt 3: Which instruction is most natural (ambiguous)?

Step 3: Build the main task

Learning objective: Understand the difference between a formal language and a natural language

Task A: Sort the statements

Tick Formal or Natural for each statement.

#StatementChoose
1 Set speed = 5 and repeat the move command 3 times.
2 Walk quickly to the door and stop nearby.
3 If the temperature > 18, turn the heater off.
4 Make the screen brighter.

Task B: Make it formal

Rewrite the instruction as short numbered steps.

"Make a hot chocolate and tidy the table."

Remove vague words like “some”, “a bit”, or “quickly”.

Task C: Test the wording

Swap instructions with a partner. Mark anything unclear, then improve it.

CheckYes / Needs work
Any ambiguous words spotted?
Write a clearer version of one unclear step

Step 4: Reflection

Encourage pupils to note what was easy, what was tricky, and what they would change.

Teacher guidance (answer prompts)

  • Quick check 1: Python code
  • Quick check 2: It is precise and unambiguous
  • Quick check 3: "Move forward a bit"
  • Key ideas: formal language has fixed rules/symbols; natural language is flexible and can be ambiguous.
  • Common misconception: "formal" means polite. Prompt: "Can a computer guess what you mean?"
