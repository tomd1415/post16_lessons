Lesson plan

Overview

Students explore the idea of precision in computing languages, contrast source vs machine code, and map the core phases of software development before drafting a simple game specification.

Timing: ~60 minutes

Resources: Laptops with browser, local Python (optional), paper/whiteboards, projector.

Vocabulary: formal language, natural language, source code, machine code, compiler, analysis, design, programming, testing, enhancement, specification.

Learning objectives

  • Understand the difference between a formal language and a natural language.
  • Define code; distinguish between source code and machine code.
  • Understand program description and specification.
  • Recognise typical activities: analysis, design, programming, testing, enhancement.

Lesson structure (suggested)

  1. Starter (8m): Natural vs formal instructions demo (ambiguous vs precise) → Activity a01 quick checks.
  2. Input (10m): Source vs machine code mini-show-and-tell (a02 quick checks).
  3. Guided practice (15m): Order the software development steps (a03 tasks A–C).
  4. Main task (20m): Draft a simple game specification (a04 tasks A–C) with peer review.
  5. Reflection (5m): Share one acceptance check and one risk per pair.

Assessment & evidence

  • Quick check responses in a01–a04 (auto-marked).
  • Notes in Step 2 fields (definitions in pupils’ own words).
  • Checklist completion and structured notes for the game specification.
  • Teacher-mode review of reflection answers.

Differentiation

  • Support: Provide sentence starters for definitions; pair a novice with a peer coach.
  • Stretch: Ask for a real opcode example in a02; request performance and accessibility constraints in a04.
  • Misconceptions to watch: “Comments are machine code”; “Testing only happens after release”.

Homework / extension

  • Find one historical example of a compiler and note what language it translated.
  • Rewrite a vague game rule into a testable requirement.
