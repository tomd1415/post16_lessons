Printable cards

Vocabulary (cut-out)

Source code

Human-readable instructions (e.g., Python).

Machine code

Binary instructions executed by the CPU.

Compiler

Translates source to machine code.

Specification

Clear, testable statement of what to build.

Analysis

Understand the problem and constraints.

Design

Plan structure (flowchart, pseudocode).

Programming

Write and assemble the code.

Testing

Check behaviour against requirements.

Enhancement

Improve after release using feedback.

Specification prompts

Goal

What does “win” look like?

Inputs

Keys, mouse/touch, start/reset.

Outputs

Screen, sound, score display.

Rules

What can/can’t happen?

Constraints

Performance, platform, accessibility.

Acceptance check

“The game should…” statements to test.

Print single-sided; cut along card edges. Use as table prompts or mini-sort tasks.
