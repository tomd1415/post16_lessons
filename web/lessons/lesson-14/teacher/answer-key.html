Answer guidance

Quick check answers

  • Tower of Hanoi: a function calls itself; base case stops recursion; each call moves closer to the base case.
  • Recursive Pattern: base case stops; each call reduces problem size; recursion can replace some loops.

Activity guidance

  • Tower of Hanoi: order is move n-1 to spare, move largest, move n-1 to target; moves count 1→1, 3→7, 4→15.
  • Recursive Pattern: look for explicit base case, depth change, and outputs for multiple depths; expect planned signature and reasoning on how recursion terminates.
