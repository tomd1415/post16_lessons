# Pupil Interface

Interface specifications for the pupil assessment experience.

## Design Principles

1. **Clarity**: Simple, uncluttered interface that focuses attention on the question
2. **Encouragement**: Language and design that builds confidence
3. **Progress**: Always show where the pupil is in the assessment
4. **Support**: Hints and scaffolds available without stigma
5. **Accessibility**: Works for all pupils, including those with additional needs

## Accessibility Features

### Keyboard Navigation
- Tab through all interactive elements
- Enter/Space to select options
- Arrow keys to navigate within question options
- Escape to close modals

### Screen Reader Support
```html
<div role="main" aria-label="Question 1 of 15">
  <h1 id="question-text">What number system do computers use?</h1>
  <div role="radiogroup" aria-labelledby="question-text">
    <label>
      <input type="radio" name="answer" value="a" aria-describedby="option-a">
      <span id="option-a">Decimal (base 10)</span>
    </label>
    ...
  </div>
</div>
```

### High Contrast Mode
- Respects `prefers-contrast: high` media query
- Additional focus indicators
- Larger text options

### Reduced Motion
- Respects `prefers-reduced-motion` media query
- Skip animations for progress bars, etc.

## Mobile Considerations

While primarily designed for Chromebooks, ensure:
- Touch targets minimum 44x44px
- No hover-only interactions
- Scrollable code editors on small screens
- Pinch-to-zoom not disabled
